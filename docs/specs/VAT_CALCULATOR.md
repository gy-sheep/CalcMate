# 부가세 계산기 스펙

> **Phase**: `Phase 4`
> **한 줄 설명**: 공급가액 또는 합계 금액을 입력하면 부가세(10%)를 자동 계산하는 부가세 계산기

---

## 개요

사용자가 "부가세 별도"(공급가액 입력) 또는 "부가세 포함"(합계 입력) 모드를 선택한 뒤 금액을 입력하면,
부가세(10%)와 나머지 금액을 자동으로 계산하여 공급가액 / 부가세 / 합계 세 항목을 실시간으로 표시한다.
수식 입력(`100000 + 50000 =`)을 지원하여 여러 항목의 금액을 합산한 뒤 부가세를 계산할 수 있다.
모든 계산은 로컬에서 수행되므로 네트워크 연결이 필요 없다.

## 기능 요구사항

### 모드 선택

- 화면 상단에 세그먼트 컨트롤로 **부가세 별도** / **부가세 포함** 두 모드를 전환한다
- **부가세 별도** (기본값): 공급가액을 입력 → 부가세, 합계 계산
- **부가세 포함**: 합계를 입력 → 공급가액, 부가세 역산
- 모드 전환 시 입력값과 수식을 유지한다 (입력한 숫자를 새 모드 기준으로 재계산)

### 금액 표시

- 3개 금액 행을 표시한다: **공급가액**, **부가세 (10%)**, **합계**
- 활성 행(입력 중인 행)은 밑줄 하이라이트(흰색)로 구분한다
- 비활성 행은 밑줄(흰색 38%)로 구분하고 계산 결과를 표시한다
- 부가세 행은 항상 계산 결과만 표시한다 (직접 입력 불가)
- 각 행의 금액 우측에 `원` 단위를 표시한다

### 금액 입력

- 키패드는 5×4 레이아웃 (환율 계산기와 동일)
- 숫자(`0-9`, `00`), 연산자(`÷ × - +`), `%`, `.`, `⌫`, `AC/C`, `=`
- 활성 행만 편집 가능하다
- 수식 입력을 지원한다 (`EvaluateExpressionUseCase` 재사용)
- `=`를 누르면 수식을 평가하고, 그 결과값을 기준으로 부가세를 계산한다
- **자릿수 제한**: 정수부 최대 12자리, 소수부 최대 8자리. 초과 시 토스트로 안내
- `00` 키: 선행 0 방지 (입력이 `0`이면 무시), 자릿수 제한에 따라 0을 1개 또는 2개 추가
- 입력값에는 천 단위 콤마가 자동 적용되어 표시
- **AC/C 동적 전환**: 입력 중이면 `C` (현재 입력만 삭제), 초기 상태이면 `AC` (전체 초기화)

### 계산

- 입력값이 변경될 때마다 즉시 부가세와 나머지 금액을 계산하여 표시한다
- 수식이 완성되지 않은 경우(연산자로 끝나는 경우)에는 현재까지 평가 가능한 값을 기준으로 계산한다
- 부가세율은 **10%** 고정이다

#### 부가세 별도 모드 (공급가액 입력)

| 항목 | 계산식 |
|------|--------|
| 공급가액 | 입력값 |
| 부가세 | 공급가액 × 0.1 |
| 합계 | 공급가액 + 부가세 |

#### 부가세 포함 모드 (합계 입력)

| 항목 | 계산식 |
|------|--------|
| 합계 | 입력값 |
| 공급가액 | ⌊합계 / 1.1⌋ (원 미만 절사) |
| 부가세 | 합계 − 공급가액 |

> **참고**: 부가세 포함 모드에서 공급가액은 원 미만 절사(floor)를 적용한다. 이는 한국 세법상 공급가액 원 미만 절사 관행을 따른 것이다.

### 결과 표시 규칙

| 조건 | 표시 형식 |
|------|-----------|
| 1,000 이상 | 천 단위 콤마 + 정수 표시 (예: `1,100,000`) |
| 1 이상 1,000 미만 | 정수 또는 소수점 이하 유효숫자 표시 (예: `100`, `99.1`) |
| 0 | `0` |
| 음수 | 부호 포함 표시 (예: `-100,000`) |

- 결과값의 소수점 이하가 0이면 정수로 표시한다 (예: `100,000.0` → `100,000`)
- 불필요한 후행 0은 제거한다 (예: `1,234.50` → `1,234.5`)

## 예외 / 엣지 케이스

| 상황 | 기대 동작 |
|------|-----------|
| 입력값이 0 | 공급가액, 부가세, 합계 모두 `0` 표시 |
| 음수 입력 (수식 결과가 음수) | 음수 기준으로 정상 계산 (예: 공급가액 `-100,000` → 부가세 `-10,000`, 합계 `-110,000`) |
| 수식 오류 (예: `0 ÷ 0`) | 입력 행에 `오류` 표시, 나머지 행 `0` 표시 |
| 매우 큰 금액 입력 | 자릿수 제한(정수 12자리)에 의해 입력 차단 |
| 소수점 입력 (예: `1,000.5`) | 부가세 별도: 부가세 `100.05`, 합계 `1,100.55` / 부가세 포함: 공급가액 원 미만 절사 적용 |
| 모드 전환 시 수식이 미완성인 경우 | 수식 상태를 유지하고 모드만 전환하여 재계산 |
| `=` 반복 입력 | 기본 계산기와 동일하게 마지막 연산 반복 |

## 비기능 요구사항

- **성능**: 입력값 변경 시 계산 결과 즉시 업데이트 (16ms 이내)
- **오프라인**: 네트워크 불필요 (모든 계산이 로컬에서 수행)
- **접근성**: 각 금액 행의 라벨(공급가액, 부가세, 합계)을 명확히 표시하여 가독성 확보
