# 부가세 계산기 스펙

> **Phase**: `Phase 4`
> **한 줄 설명**: 공급가액 또는 합계 금액을 입력하면 부가세를 자동 계산하는 부가세 계산기

---

## 개요

사용자가 "부가세 별도"(공급가액 입력) 또는 "부가세 포함"(합계 입력) 모드를 선택한 뒤 금액을 입력하면,
부가세와 나머지 금액을 자동으로 계산하여 공급가액 / 부가세 / 합계 세 항목을 실시간으로 표시한다.
세율은 기기 로케일 기반으로 자동 설정되며(기본 10%), 사용자가 직접 변경할 수 있다.
수식 입력(`100000 + 50000 =`)을 지원하여 여러 항목의 금액을 합산한 뒤 부가세를 계산할 수 있다.
모든 계산은 로컬에서 수행되므로 네트워크 연결이 필요 없다.

**테마**:
- **배경**: 다크 그라디언트 (구현 시 별도 선정 예정)
- **영수증 카드**: 크림색 (`#F5F5F0`) 배경 + 그림자(BoxShadow)로 부유 효과

- **톱니 모양**: `CustomClipper` 또는 `ClipPath`로 구현
- **점선 구분선**: `CustomPaint`로 구현

## 기능 요구사항

### 모드 선택

- 화면 상단에 세그먼트 컨트롤로 **부가세 별도** / **부가세 포함** 두 모드를 전환한다
- **부가세 별도** (기본값): 공급가액을 입력 → 부가세, 합계 계산
- **부가세 포함**: 합계를 입력 → 공급가액, 부가세 역산
- 모드 전환 시 입력값과 수식을 유지하고 새 모드 기준으로 재계산한다

### 입력 영역

- 화면 상단에 별도의 입력 영역을 배치한다 (결과 영역과 시각적으로 분리)
- 입력값은 큰 폰트로 우측 정렬하여 표시한다
- 수식이 있는 경우 입력값 아래에 작은 폰트로 수식을 표시한다

### 결과 영역 (영수증 스타일)

- 입력 영역 아래에 결과 카드를 배치한다
- 3개 항목을 나열한다: **공급가액**, **부가세 ({세율}%) ⓘ**, **합계**
- 합계 위에 굵은 구분선을 표시한다
- **합계**는 다른 항목보다 큰 폰트 또는 bold로 강조한다
- 각 행: 좌측에 라벨, 우측에 금액 + `원` 표시

### 세율 설정

#### 기본 세율 (로케일 기반 자동 설정)

- 앱 최초 실행 시 기기의 국가 설정(Locale)을 읽어 기본 세율을 자동 적용한다
- 매핑 테이블에 없는 국가는 **10%**를 fallback으로 사용한다
- 세율 테이블은 앱에 하드코딩하며, 각국 세율 변경 시 앱 업데이트로 반영한다

| 국가 코드 | 국가 | 세율 |
|----------|------|------|
| KR | 대한민국 | 10% |
| JP | 일본 | 10% |
| GB | 영국 | 20% |
| DE | 독일 | 19% |
| FR | 프랑스 | 20% |
| IT | 이탈리아 | 22% |
| ES | 스페인 | 21% |
| AU | 호주 | 10% |
| CA | 캐나다 | 5% |

#### 세율 변경

- 결과 영역의 `부가세 (10%)` 라벨에서 **세율 부분을 탭**하면 세율 편집 모드로 진입한다
- 세율 편집 시 키패드 입력이 세율 필드로 전환된다
- **세율 범위**: 0% ~ 99% (정수만 입력 가능)
- `=` 또는 결과 영역 외부 탭 시 세율 편집 모드를 종료하고 금액 입력으로 복귀한다
- 변경된 세율은 즉시 반영되어 결과를 재계산한다
- 세율은 **두 모드(별도/포함) 모두에서 변경 가능**하다

### 세율 참고표 (ⓘ 버튼)

- `부가세 ({세율}%) ⓘ` 라벨의 **ⓘ 버튼**을 탭하면 세율 참고 Bottom Sheet가 열린다
- 각국 표준 VAT/소비세율을 국가별로 표시한다
- **표시 대상**: 부가세·소비세·GST 등 단일 세율 기반 항목만 포함 (개별소비세 등 복합 과세 구조 제외)
- 미국은 연방 VAT가 없으므로 "주마다 상이 (0~10.25%)" 로 안내한다

### 금액 입력

- 키패드는 5×4 레이아웃 (환율 계산기와 동일)
- 숫자(`0-9`, `00`), 연산자(`÷ × - +`), `%`, `.`, `⌫`, `AC`, `=`
- 수식 입력을 지원한다 (`EvaluateExpressionUseCase` 재사용)
- `=`를 누르면 수식을 평가하고, 그 결과값을 기준으로 부가세를 계산한다
- **자릿수 제한**: 정수부 최대 12자리, 소수부 최대 8자리. 초과 시 토스트로 안내
- `00` 키: 선행 0 방지 (입력이 `0`이면 무시), 자릿수 제한에 따라 0을 1개 또는 2개 추가
- 입력값에는 천 단위 콤마가 자동 적용되어 표시
- **AC**: 입력값 전체를 초기화하여 `0`으로 돌아간다. 항상 `AC`로 표시한다 (C 전환 없음)

### 계산

- 입력값이 변경될 때마다 즉시 부가세와 나머지 금액을 계산하여 표시한다
- 수식이 완성되지 않은 경우(연산자로 끝나는 경우)에는 현재까지 평가 가능한 값을 기준으로 계산한다

#### 부가세 별도 모드 (공급가액 입력)

| 항목 | 계산식 |
|------|--------|
| 공급가액 | 입력값 |
| 부가세 | 공급가액 × (세율 / 100) |
| 합계 | 공급가액 + 부가세 |

#### 부가세 포함 모드 (합계 입력)

| 항목 | 계산식 |
|------|--------|
| 합계 | 입력값 |
| 공급가액 | ⌊합계 / (1 + 세율/100)⌋ (원 미만 절사) |
| 부가세 | 합계 − 공급가액 |

> **참고**: 부가세 포함 모드에서 공급가액은 원 미만 절사(floor)를 적용한다. 이는 한국 세법상 공급가액 원 미만 절사 관행을 따른 것이다.

### 결과 표시 규칙

| 조건 | 표시 형식 |
|------|-----------|
| 1,000 이상 | 천 단위 콤마 + 정수 표시 (예: `1,100,000`) |
| 1 이상 1,000 미만 | 정수 또는 소수점 이하 유효숫자 표시 (예: `100`, `99.1`) |
| 0 | `0` |
| 음수 | 부호 포함 표시 (예: `-100,000`) |

- 결과값의 소수점 이하가 0이면 정수로 표시한다 (예: `100,000.0` → `100,000`)
- 불필요한 후행 0은 제거한다 (예: `1,234.50` → `1,234.5`)

## 예외 / 엣지 케이스

| 상황 | 기대 동작 |
|------|-----------|
| 입력값이 0 | 공급가액, 부가세, 합계 모두 `0` 표시 |
| 음수 입력 (수식 결과가 음수) | 음수 기준으로 정상 계산 (예: 공급가액 `-100,000` → 부가세 `-10,000`, 합계 `-110,000`) |
| 수식 오류 (예: `0 ÷ 0`) | 입력 영역에 `오류` 표시, 결과 영역 `0` 표시 |
| 매우 큰 금액 입력 | 자릿수 제한(정수 12자리)에 의해 입력 차단 |
| 소수점 입력 (예: `1,000.5`) | 부가세 별도: 부가세 `100.05`, 합계 `1,100.55` / 부가세 포함: 공급가액 원 미만 절사 적용 |
| 모드 전환 시 수식이 미완성인 경우 | 수식 상태를 유지하고 모드만 전환하여 재계산 |
| `=` 반복 입력 | 기본 계산기와 동일하게 마지막 연산 반복 |
| 세율 0% 입력 | 부가세 `0`, 합계 = 공급가액 |
| 세율 편집 중 빈 값 | 세율을 0%로 처리 |
| 로케일 매핑 없는 국가 | 세율 10% fallback 적용 |

## 비기능 요구사항

- **성능**: 입력값 변경 시 계산 결과 즉시 업데이트 (16ms 이내)
- **오프라인**: 네트워크 불필요 (모든 계산이 로컬에서 수행)
- **접근성**: 각 금액 행의 라벨(공급가액, 부가세, 합계)을 명확히 표시하여 가독성 확보
- **세율 갱신**: 각국 세율 변경 시 앱 업데이트로 반영 (Remote Config 미사용)
