# 부가세 계산기 — 입력 모드 UX 개선 기획서

> **Phase**: `Phase 4 수정`
> **한 줄 설명**: 인라인 토글로 입력 모드를 전환하고, 모드에 따라 결과 카드 행 순서와 강조를 변경

---

## 배경 / 문제 인식

기존 체크박스("부가세 별도") 방식의 문제:

1. **라벨 의미 불명확**: "부가세 별도"는 가격표 수식어로, 입력 모드를 나타내는 UI 라벨로 직관적이지 않음
2. **결과 카드 구조 불일치**: 두 모드 모두 동일한 행 순서(공급가액 → 부가세 → 합계)로 표시되어, 합계금액 입력 시 사용자가 알고 싶은 공급가액이 강조되지 않음

---

## 개선 내용

### 1. 체크박스 → 인라인 토글로 교체

체크박스 자리에 두 옵션을 나란히 표시하는 토글로 교체한다.

```
공급가액  |  합계금액
```

- 선택된 옵션: 진한 색상(`kVatReceiptText`) + w600
- 미선택 옵션: 흐린 색상(`kVatReceiptSecondary`) + w400
- 탭 시 즉시 모드 전환 및 재계산
- 모드 전환 시 입력값·수식 유지

### 2. 결과 카드 — 모드에 따라 행 순서·강조 변경

항상 세 값(공급가액·부가세·합계)을 모두 표시하되, 모드에 따라 순서와 강조 행이 달라진다.

**공급가액 모드** (사용자가 공급가액 입력):

```
공급가액:    100,000원   ← 입력값
부가세(10%):  10,000원
─────────────────────
합계:        110,000원   ← 강조 (계산된 값)
```

**합계금액 모드** (사용자가 합계 입력):

```
합계:        110,000원   ← 입력값
부가세(10%):  10,000원
─────────────────────
공급가액:    100,000원   ← 강조 (계산된 값)
```

## 기능 요구사항

### 인라인 토글 (`VatModeToggle`)

- 점선 구분선 사이에 위치 (기존 체크박스 자리)
- 좌측 정렬 (아래 결과 행 라벨과 x축 일치)
- 두 옵션(`공급가액` / `합계금액`)을 세로 구분선(`|`)으로 분리
- 탭 시 즉시 모드 전환

### 결과 카드

| 모드 | 첫 번째 행 | 두 번째 행 | 강조 행 |
|------|-----------|-----------|--------|
| 공급가액 | 공급가액 (입력값) | 부가세 | 합계 |
| 합계금액 | 합계 (입력값) | 부가세 | 공급가액 |

- 강조 행: 실선 구분선 아래, `textStyleResult18` / `textStyleResult22` bold
- 일반 행: `textStyleCaption` / `kVatReceiptSecondary`

### 계산 로직 (기존 유지)

| 모드 | 입력 | 계산 |
|------|------|------|
| 공급가액 | 공급가액 | 부가세 = 공급가액 × 세율, 합계 = 공급가액 + 부가세 |
| 합계금액 | 합계 | 공급가액 = ⌊합계 / (1 + 세율/100)⌋, 부가세 = 합계 − 공급가액 |
