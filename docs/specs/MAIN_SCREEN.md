# 메인 화면 기획서

> **Phase**: `Phase 0 (기초) + 지속 확장`
> **한 줄 설명**: 13개 계산기를 카드 목록으로 탐색하고 진입하는 앱의 홈 화면

---

## 개요

CalcMate의 첫 진입 화면. 사용 가능한 계산기를 카드 목록으로 표시하며,
카드 탭으로 해당 계산기로 진입한다. 순서 편집, 카드 숨기기 등 목록 개인화 기능을 제공한다.

---

## 화면 구성

```
┌───────────────────────────────────┐
│  CalcMate                    ⋮    │  AppBar (투명, 스크롤 시 페이드 아웃)
├───────────────────────────────────┤
│  ┌─────────────────────────────┐  │
│  │  계산기 카드                 │  │  CalcModeCard (배경 이미지 + 아이콘)
│  └─────────────────────────────┘  │
│  ┌─────────────────────────────┐  │
│  │  계산기 카드  [숨기기]       │  │  ← 좌측 스와이프 시 버튼 노출
│  └─────────────────────────────┘  │
│  ...                              │
└───────────────────────────────────┘
```

---

## 기능 요구사항

### 일반 모드

- 표시 중인 계산기(`isVisible == true`)를 카드 목록으로 렌더링
- 카드 탭 → Container Transform 애니메이션으로 계산기 화면 진입
- 스크롤 시 AppBar 페이드 아웃 + 상태바 영역 블러 오버레이
- ⋮ 메뉴 → **순서 편집** / **설정** 드롭다운

### 순서 편집 모드

- ⋮ 메뉴 → 순서 편집 탭 시 진입
- `ReorderableListView`로 드래그 정렬
- 카드 우측에 드래그 핸들 아이콘 표시
- AppBar "완료" 버튼으로 종료 및 순서 저장 (`SharedPreferences` — `'calc_entry_order'`)
- 편집 모드 중 카드 스와이프 비활성화

### 카드 스와이프 숨기기

- 일반 모드에서 카드를 **좌측 스와이프** 시 "숨기기" 버튼 노출
- "숨기기" 탭 → 해당 계산기를 목록에서 제거 (슬라이드 아웃 애니메이션)
- 내부적으로 `CalcModeEntry.isVisible = false` 처리
- `MainScreenViewModel.toggleVisibility(id)` 호출 — 설정 > 계산기 관리와 동일 로직 공유
- **최소 1개 표시 유지**: 목록에 카드가 1개만 남으면 스와이프 제스처 비활성화
- 숨긴 계산기는 **설정 > 계산기 관리**에서만 다시 표시 가능
- 편집 모드(순서 편집) 중에는 스와이프 비활성화

---

## 예외 / 엣지 케이스

| 상황 | 기대 동작 |
|------|-----------|
| 카드가 1개만 남은 상태에서 좌측 스와이프 | 스와이프 제스처 무반응 (비활성) |
| 모든 계산기가 숨겨진 상태로 앱 재시작 | 전체 표시 상태로 복구 (방어 처리) |
| 편집 모드 중 스와이프 시도 | 무반응 |
| 저장된 순서에 없는 신규 계산기 추가 시 | 목록 맨 뒤에 자동 추가 |

---

## 상태 관리

- **ViewModel**: `MainScreenViewModel` (`NotifierProvider`)
- **State**: `MainScreenState` (entries, isScrolled, isEditMode)
- **저장소**: `SharedPreferences`
  - `'calc_entry_order'` — 계산기 순서 (StringList)
  - `'calc_entry_hidden'` — 숨긴 계산기 id 목록 (StringList)
