# 기본 계산기 스펙

> **Phase**: `Phase 1`
> **한 줄 설명**: 사칙연산과 괄호, 퍼센트/나머지 연산을 지원하는 기본 계산기

---

## 개요

사용자가 숫자와 연산자를 순서대로 입력하면 수식을 조합하고, `=`를 누르는 시점에 결과를 계산하여 표시한다.
괄호를 통한 수식 그룹핑, `%`의 컨텍스트 기반 퍼센트/나머지 연산, `=` 반복 등을 지원한다.

## 기능 요구사항

### 숫자 입력

- 숫자 버튼(0~9) 탭 시 현재 입력값에 해당 숫자를 추가한다
- 초기 상태(`0`)에서 숫자를 누르면 `0`을 대체한다 (`01` 방지)
- `)` 바로 뒤에서 숫자를 누르면 암묵적 곱셈을 삽입한다 (예: `(3+2)` → `5` → `(3+2)×5`)
- 계산 완료 상태(`isResult`)에서 숫자를 누르면 새 수식으로 시작한다

### 연산자 입력 (`÷ × - +`)

- 현재 입력값 끝에 연산자를 추가하여 수식을 이어간다
- 연산자가 이미 맨 끝에 있으면 새 연산자로 교체한다 (연속 입력 시 마지막 연산자 유효). 단, 같은 연산자를 다시 누르면 무시한다
- 연산자 뒤에서 `-`를 누르면 음수 입력으로 처리한다 (예: `5×` → `-` → `5×-`). 연산자 교체가 아닌 음수 대기 상태 진입
- 음수 대기 상태(`5×-`)에서 다른 연산자를 누르면 음수 입력을 취소하고 연산자를 교체한다 (예: `5×-` → `+` → `5+`)
- 초기 상태(`0`)에서 `-`를 누르면 음수 입력 모드로 진입한다 (`-` 표시). 나머지 연산자(`+ × ÷`)는 무시한다
- `(` 바로 뒤에서는 `-`만 허용한다 (음수 입력, 예: `(-5`). 나머지 연산자(`+ × ÷`)는 무시한다
- 소수점으로 끝나는 상태(`5.`)에서 연산자를 누르면 `.`을 유지한 채 연산자를 추가한다 (예: `5.+`). 계산 시점에 `5.`은 `5`로 처리한다
- 계산 완료 상태에서 연산자를 누르면 결과값에 이어 수식을 이어간다

### 소수점 (`.`)

- 현재 숫자 세그먼트에 소수점이 없을 때만 추가한다 (중복 방지)
- 연산자 바로 뒤 또는 `(` 바로 뒤에서 `.`를 누르면 `0.`으로 시작한다
- 음수 대기 상태(`-` 또는 `5×-`)에서 `.`를 누르면 `-0.`으로 시작한다 (예: `5×-` → `.` → `5×-0.`)
- `)` 바로 뒤에서 `.`를 누르면 암묵적 곱셈 + `0.`으로 시작한다 (예: `(3+2)` → `.` → `(3+2)×0.`)
- `%` 바로 뒤에서 `.`를 누르면 암묵적 곱셈 + `0.`으로 시작한다 (예: `50%` → `.` → `50%×0.`)
- 계산 완료 상태에서 `.`를 누르면 `0.`으로 새 입력을 시작한다

### 괄호 (`()`)

문맥에 따라 여는 괄호 `(`와 닫는 괄호 `)`를 자동으로 판단하는 스마트 괄호 버튼이다.

**`(` 삽입 조건** — 다음 상황에서 버튼을 누르면 `(`를 삽입한다:
- 초기 상태(`0`)에서 누른 경우 → `(`로 대체
- 연산자(`+ - × ÷`) 바로 뒤에서 누른 경우 → `(` 추가
- 여는 괄호 `(` 바로 뒤에서 누른 경우 → `(` 추가 (중첩 괄호)
- 숫자 바로 뒤에서 누른 경우 → `×(` 삽입 (암묵적 곱셈)
- 계산 완료 상태에서 누른 경우 → `(` 로 새 수식 시작

**`)` 삽입 조건** — 다음 조건을 **모두** 만족할 때 `)`를 삽입한다:
- 닫히지 않은 `(`가 하나 이상 존재한다
- 마지막 문자가 숫자 또는 `)`이다 (연산자 뒤에서는 닫지 않는다)

**입력 무시 조건**:
- 닫히지 않은 `(`가 없고, `)` 삽입 조건도 아닌 경우

**입력 예시**:

| 현재 입력 | 버튼 | 결과 | 설명 |
|-----------|------|------|------|
| `0` | `()` | `(` | 초기 상태 → 여는 괄호 |
| `(` | `()` | `((` | 여는 괄호 뒤 → 중첩 |
| `(5+3` | `()` | `(5+3)` | 숫자 뒤 + 미닫힌 괄호 → 닫기 |
| `(5+` | `()` | `(5+(` | 연산자 뒤 → 여는 괄호 |
| `5` | `()` | `5×(` | 숫자 뒤 → 암묵적 곱셈 + 여는 괄호 |
| `(3+2)` | `()` | `(3+2)×(` | 닫는 괄호 뒤 → 암묵적 곱셈 + 여는 괄호 |

### 퍼센트 / 나머지 연산 (`%`)

`%`는 문맥에 따라 퍼센트 또는 나머지 연산(mod)으로 동작한다.

**퍼센트** — `%` 뒤에 연산자 또는 `=`이 오는 경우:
- `A + B%` → `A + (A × B / 100)` (앞 숫자 기준 퍼센트)
- `A - B%` → `A - (A × B / 100)`
- `A × B%` 또는 `A ÷ B%` 또는 단독 `B%` → `B ÷ 100`

**나머지 연산(mod)** — `%` 뒤에 바로 숫자가 오는 경우:
- `A%B` → `A mod B` (예: `50%3` = `2`)

**입력 규칙**:
- 현재 입력 끝에 `%` 기호를 추가한다
- `(` 바로 뒤에서 `%`를 누르면 `0%`로 시작한다
- `)` 바로 뒤에서 `%`를 누르면 허용한다 (예: `(3+2)%`)
- 계산 완료 상태에서 `%`를 누르면 결과값에 `%`를 붙이고 수식 표시 영역은 초기화한다 (예: 결과 `8` → `8%`)
- 연산자 바로 뒤에서 `%`를 누르면 연산자를 `%`로 교체한다 (예: `5+` → `%` → `5%`)
- 이미 `%`가 있을 때 다시 `%`를 누르면 **마지막 피연산자+%** 만 괄호로 감싸고 `%`를 재적용한다 (예: `5%` → `(5%)%` → `((5%)%)%`, `5×9%` → `5×(9%)%`)
- 음수 대기 상태(`5×-`)에서 `%`를 누르면 무시한다

### 백스페이스 (`⌫`)

- 마지막 입력 문자를 한 글자씩 제거한다
- 입력값이 한 자리이면 `0`으로 초기화한다
- 계산 완료 상태에서 `⌫`를 누르면 전체 초기화(`0`)한다

### 클리어 (`AC`)

- 입력값 전체를 초기화하여 `0`으로 돌아간다
- 항상 `AC`로 표시한다 (C 전환 없음)

### 계산 (`=`)

- 수식을 평가하여 결과를 표시하고, 수식은 작게 위에 표시한다
- 연산자가 없는 상태(숫자만 입력)에서 `=`를 누르면 무시한다
- 불완전한 수식에서 `=`를 누르면 무시한다: 연산자로 끝나는 경우, 음수 대기 상태, `(`만 있는 경우
- 결과 상태에서 `=`를 반복하면 마지막 연산자와 피연산자를 재적용한다 (반복 `=`)
  - 예: `5 + 3 =` → `8`, 다시 `=` → `11`, 다시 `=` → `14`

## 표시 규칙

- **천 단위 콤마**: 숫자에 자동 적용 (예: `1234567` → `1,234,567`)
- **음수 표시**: 음수 대기 입력(`5×-3`)은 그대로 표시한다 (괄호 자동 감싸기 없음)
- **동적 폰트**: 숫자가 길어질수록 폰트 크기 자동 축소 (최대 80pt, 최소 10pt)
- **입력값/결과 스크롤**: 폰트 축소 한계 초과 시 오른쪽 기준 가로 스크롤로 전체 내용 확인 가능
- **수식 표시 스크롤**: 수식이 한 줄 너비를 초과하면 연산 기호 앞에서 줄바꿈하여 최대 2줄로 표시하고, 그래도 넘치면 오른쪽 기준 가로 스크롤 적용

## 예외 / 엣지 케이스

| 상황 | 기대 동작                       |
|------|-----------------------------|
| `0 ÷ 0` | `정의되지 않음` 표시                |
| 괄호가 닫히지 않은 상태에서 `=` | 자동으로 부족한 `)` 를 채운 뒤 계산      |
| 매우 긴 소수 결과 | 불필요한 trailing 0 제거, 소수점 최대 9자리 |
| `-0` 상태 | `0`으로 표시                    |
| 결과가 정수인 경우 | 소수점 없이 표시 (예: `3.0` → `3`)  |
