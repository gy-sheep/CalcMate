# BMI 계산기 기획서

> **Phase**: `Phase 9`
> **한 줄 설명**: 키·몸무게를 슬라이더로 입력하면 BMI와 건강 범주를 시각적 게이지로 즉시 확인하는 계산기

---

## 개요

BMI(Body Mass Index)는 키와 몸무게로 산출하는 체질량지수로, 자신의 체중이 건강 범위 안에 있는지 빠르게 확인하고 싶을 때 사용한다.
단순히 숫자만 보여주는 것이 아니라 **"나는 어디쯤 있는가"** 를 직관적으로 느낄 수 있도록 게이지 중심의 UI를 채택한다.
또한 현재 키 기준으로 **건강 체중 범위**를 역산해 보여줌으로써, 결과가 단순한 수치 확인에서 그치지 않고 실질적인 건강 정보로 이어지도록 한다.
**글로벌 앱**이므로 단위계(미터법·야드파운드법)와 BMI 판정 기준(WHO 글로벌·아시아태평양)을 device locale에 따라 자동 적용한다.

## 글로벌 대응 (locale 자동 감지)

### 단위계 기본값

| locale 국가코드 | 기본 단위 |
|----------------|----------|
| `US` | lb · ft·in (야드파운드법) |
| 그 외 전체 | kg · cm (미터법) |

- 앱 첫 진입 시 device locale의 countryCode를 읽어 기본값 자동 설정.
- 사용자가 AppBar 토글로 언제든지 수동 전환 가능.
- **마지막 선택값은 SharedPreferences에 저장**해 재진입 시 유지.
- 내부 저장은 항상 kg·cm 기준. 표시 시에만 변환.

### BMI 판정 기준

| locale 국가코드 | 적용 기준 | 범주 수 |
|----------------|----------|--------|
| KR·JP·CN·TW·HK·MO·SG·MY·TH·VN·PH·ID·KH·MM·BD | WHO 아시아태평양 (대한비만학회 준용) | 5범주 |
| 그 외 전체 | WHO 글로벌 | 4범주 |

#### WHO 글로벌 기준 (4범주)

| 범주 | BMI 범위 | 색상 |
|------|---------|------|
| 저체중 | < 18.5 | 파랑 #5B9BD5 |
| 정상 | 18.5 – 24.9 | 초록 #4CAF50 |
| 과체중 | 25.0 – 29.9 | 주황 #FF9800 |
| 비만 | ≥ 30.0 | 빨강 #F44336 |

#### WHO 아시아태평양 기준 (5범주)

| 범주 | BMI 범위 | 색상 |
|------|---------|------|
| 저체중 | < 18.5 | 파랑 #5B9BD5 |
| 정상 | 18.5 – 22.9 | 초록 #4CAF50 |
| 과체중 | 23.0 – 24.9 | 노랑 #FFCC02 |
| 비만 1단계 | 25.0 – 29.9 | 주황 #FF9800 |
| 비만 2단계 | ≥ 30.0 | 빨강 #F44336 |

- 게이지 아크 구간 수 및 색상 배치는 적용 기준에 따라 자동 변경.
- 범주 그리드는 5범주일 때 **3+2 두 행** 레이아웃으로 자동 전환.
- 게이지 아래 **"WHO 아시아태평양 기준 (대한비만학회 준용)"** 등 출처 배지 표시.

## 기능 요구사항

### 입력

- **키 슬라이더**: 범위 100 ~ 220 cm, 1 cm 단위.
- **몸무게 슬라이더**: 범위 20 ~ 150 kg, 0.5 kg 단위.
- 슬라이더 값 버튼을 탭하면 숫자 직접 입력 다이얼로그 표시.
  - 범위를 벗어난 값 입력 시 최소·최대값으로 클램프 + 진동 피드백.
- **단위 토글** (AppBar 우측): `kg·cm` ↔ `lb·ft·in`
  - 전환 시 현재 값을 변환하여 슬라이더와 표시 즉시 업데이트.
  - 선택값 SharedPreferences 저장 (locale 기본값 덮어쓰기).

### 계산

- **BMI** = 몸무게(kg) ÷ 키(m)²
- 내부 저장은 항상 kg·cm. 야드파운드 입력 시 변환 후 동일 공식 적용.
- 슬라이더 변경 즉시 실시간 재계산 (별도 버튼 없음).

### 결과 표시

#### 아크 게이지

- 반원(180°) 아크, locale 기준에 따라 4~5 컬러 구간 자동 적용.
- 바늘이 현재 BMI 위치를 가리키며 값 변경 시 easeOutCubic 애니메이션.
- 게이지 중앙 하단에 BMI 수치(소수 첫째 자리)와 범주 라벨 표시.
- 범주 라벨 색상은 해당 범주 색상과 일치.

#### 건강 체중 범위 카드

- 현재 키 기준, 적용 기준의 **정상 범주 BMI 범위**에 해당하는 체중 범위 역산 표시.
  - 글로벌: BMI 18.5 ~ 24.9 기준
  - 아시아태평양: BMI 18.5 ~ 22.9 기준
- 현재 체중이 범위 내이면 초록 테두리 + 확인 메시지 표시.

#### 범주 안내 그리드

- 적용 기준에 따라 4셀 또는 5셀(3+2 두 행) 레이아웃.
- 현재 범주 셀: 배경 강조 + 색상 테두리.
- 각 셀: 범주명 + BMI 범위 표시.

## UX 인터랙션 흐름

```
앱 진입
  → locale 감지 → 단위계 기본값 설정 + BMI 기준 선택
  → 기본값(170 cm, 65 kg)으로 즉시 결과 표시
  → 슬라이더 드래그 → 게이지 바늘 실시간 이동 + 수치 업데이트
  → 값 버튼 탭 → 다이얼로그 입력 → 슬라이더 + 게이지 동기화
  → 단위 토글 탭 → 값 변환, 저장
```

### 핵심 UX 원칙

1. **버튼 없는 실시간 계산**: 입력과 결과가 동시에 움직여 즉각적인 피드백.
2. **게이지 중심 설계**: "내가 어디쯤인가"를 공간적으로 인지.
3. **역산 정보 제공**: 현재 키 기준 건강 체중 범위를 자동 계산.
4. **비판단적 표현**: 경고 아이콘 없이 색상과 라벨로만 구분.
5. **locale 자동 대응**: 첫 진입부터 사용자 지역에 맞는 단위·기준 적용.

## 예외 / 엣지 케이스

| 상황 | 기대 동작 |
|------|-----------|
| 키 입력이 범위(100~220 cm) 초과 | 최솟·최대값으로 클램프 + 진동 피드백 |
| 몸무게 입력이 범위(20~150 kg) 초과 | 최솟·최대값으로 클램프 + 진동 피드백 |
| BMI가 게이지 범위(10~40) 초과 | 바늘을 끝에 고정, 수치는 정확하게 표시 |
| ft·in 입력 시 feet만 입력 | inch는 0으로 계산 |
| countryCode가 null인 경우 | 미터법 + WHO 글로벌 기준으로 폴백 |
| SharedPreferences 저장 단위와 locale 기본값 다를 경우 | 저장값 우선 적용 |

## 데이터 모델

```
BmiState {
  double heightCm          // 항상 cm로 저장
  double weightKg          // 항상 kg으로 저장
  UnitSystem unitSystem    // metric | imperial
  BmiStandard standard     // global | asian (locale 자동 감지, 수동 변경 불가)
}

BmiResult {
  double bmi
  BmiCategoryDef category  // 기준별 범주 정의 (label, rangeLabel, color, min, max)
  double healthyWeightMin  // 적용 기준 정상 범주 하한 체중
  double healthyWeightMax  // 적용 기준 정상 범주 상한 체중
  bool isInHealthyRange
}
```

## 참고 / 레퍼런스

- **WHO 글로벌 BMI 기준**: 저체중 <18.5 / 정상 18.5–24.9 / 과체중 25–29.9 / 비만 ≥30
- **WHO 아시아태평양 기준**: 저체중 <18.5 / 정상 18.5–22.9 / 과체중 23–24.9 / 비만1 25–29.9 / 비만2 ≥30
- **대한비만학회**: WHO 아시아태평양 기준 준용
- **공식**: BMI = 몸무게(kg) ÷ (키(m))²
- **단위 변환**: 1 kg = 2.20462 lb / 1 cm = 2.54 in / 1 ft = 30.48 cm
- **야드파운드법 기본 적용 국가**: 미국(US) — 일상적 체중·신장 표기에 lb·ft·in 사용
- **비판단적 건강 UI 원칙**: 색상만으로 단계 구분, 경고·공포 유발 아이콘 미사용
