# 실수령액 계산기 기획서

> **Phase**: `Phase 15`
> **한 줄 설명**: 월급 또는 연봉을 슬라이더와 정밀 조절로 시뮬레이션하며 실수령액과 공제 내역을 실시간으로 확인하는 계산기

---

## 개요

"연봉 4,500만 원이면 실제로 얼마 받지?"라는 질문에 즉시 답을 주는 계산기다.
직장인이 급여명세서를 받기 전 실수령액을 예측하거나, 취준생·이직 준비자가 연봉 협상 시 실질 소득을 파악하는 데 사용된다.
슬라이더로 빠르게 금액 구간을 탐색하고, 미세 조절 버튼 또는 직접 입력으로 정확한 금액을 설정할 수 있다.
국민연금·건강보험·장기요양·고용보험·소득세·지방소득세 6개 공제 항목이 항상 펼쳐진 상태로 표시되어 투명하게 확인할 수 있다.

---

## 사용자 시나리오

| 시나리오 | 행동 |
|----------|------|
| 이직 제안 연봉 빠른 검토 | 연봉 탭 → 슬라이더로 제안 연봉 구간 이동 → 실수령액 즉시 확인 |
| 정확한 연봉 실수령 계산 | 금액 표시 탭 → 키패드 모달에서 정확한 금액 입력 |
| 연봉 인상 효과 시뮬레이션 | 단위 선택 [100만 ▾] → [+] 반복 탭하며 변화 확인 |
| 부양가족 추가 효과 확인 | 부양가족 수 [+] → 소득세 감소폭 실시간 확인 |

### 레이아웃 구조

- **고정 상단**: 앱바 + 모드 탭
- **스크롤 영역**: 금액 입력 + 슬라이더 + 미세 조절 + 결과 + 공제 내역
- **고정 하단**: 부양가족 수 조절기

### 모드 탭 (월급 / 연봉)

- **월급 모드**: 입력값 = 월 급여 / 결과 = 월 실수령액 + 연 환산 보조 표시
  - 슬라이더 범위: 100만 ~ 1,000만 원
- **연봉 모드**: 입력값 = 연봉 / 결과 = 월 실수령액 + 연 실수령액 보조 표시
  - 슬라이더 범위: 2,400만 ~ 1억 원
- 탭 전환 시 입력 금액 자동 환산 (× 12 / ÷ 12), 슬라이더 위치도 함께 이동

### 금액 표시 + 직접 입력

- 현재 금액을 중앙에 크게 표시 (3자리 콤마 포맷)
- 우측에 편집 아이콘(✎) 표시
- **탭 시**: 키패드 모달 바텀시트 열림
  - 숫자 키패드 (7 8 9 / 4 5 6 / 1 2 3 / 00 0 ⌫)
  - 확인 버튼으로 닫기, 슬라이더 위치 금액에 맞게 이동
  - 슬라이더 범위 초과 입력 시 슬라이더는 최대/최소 끝으로 고정, 금액은 그대로 유지

### 슬라이더

- 선형 슬라이더로 금액 구간 빠르게 탐색
- 드래그 시 금액 표시 + 결과 + 공제 내역 실시간 업데이트
- 슬라이더 양 끝에 범위 최소/최대값 표시

### 미세 조절 버튼

```
[−]  [100만 ▾]  [+]
```

- `[−]` / `[+]`: 현재 선택된 단위만큼 금액 감소/증가
- `[100만 ▾]` (단위 선택 버튼): 탭 시 팝업 메뉴 표시

  ```
  ┌──────────────┐
  │  ✓  100만   │  ← 현재 선택
  │     10만    │
  │     만원    │
  │     천만    │
  └──────────────┘
  ```

  - 선택 항목: 만원 / 10만 / 100만 / 천만
  - 선택된 단위는 버튼에 표시 (예: `[10만 ▾]`)
  - 기본값: 100만
- `[−]` 탭 시 최솟값(0) 미만으로 내려가지 않음
- `[+]` 탭 시 최대 입력 제한 초과하지 않음

### 결과 영역

- "실수령액" 레이블 (좌측 정렬) + 월 실수령액 금액·단위 (우측 정렬, 큰 폰트, 골드색)
- "연 환산 XXX,XXX,XXX 원" 보조 텍스트 (월급 모드, 우측 정렬) / "연 실수령 XXX,XXX,XXX 원" (연봉 모드)
- 입력값 0일 때: `—` 표시

### 공제 내역 영역

- **항상 펼쳐진 상태** (아코디언 없음)
- 상단 구분선 + "공제 합계 XXX,XXX 원"
- 6개 항목 리스트 (항목명 좌측 / 금액 우측 정렬):
  - 국민연금
  - 건강보험
  - 장기요양보험
  - 고용보험
  - 소득세
  - 지방소득세
- 슬라이더·미세 조절·직접 입력 변경 시 모든 항목 실시간 업데이트

### 부양가족 수 조절기 (고정 하단)

- 레이블: "부양가족 수" + 툴팁 아이콘 (탭 시 "본인 포함 기준, 소득세 계산에 반영됩니다" 안내)
- `[−]` / 숫자 / `[+]` 구성
- 범위: 1 ~ 11명 / 기본값: 1명 (본인)
- 경계값에서 해당 버튼 비활성화

> **UI 스타일**: 토큰·색상·위젯 상세는 `docs/dev/SALARY_CALCULATOR_IMPL.md` 구현 명세 참조.

## 기능 요구사항

### 입력

- 슬라이더 드래그로 빠른 금액 탐색
- 미세 조절 버튼으로 단위 기반 증감
- 단위 선택: 만원 / 10만 / 100만 / 천만 (팝업 메뉴)
- 금액 표시 탭 → 키패드 모달에서 정확한 금액 직접 입력
- 모든 입력 방식에서 결과 실시간 반영
- 모드 탭 전환 시 금액 자동 환산 (× 12 / ÷ 12)
- 부양가족 수 변경 시 소득세·지방소득세 즉시 재계산

### 계산 — 4대보험

| 항목 | 공식 | 2024년 기준 |
|------|------|-------------|
| 국민연금 | min(과세급여, 기준소득월액 상한) × 4.5% | 상한 6,170,000원 → 최대 277,650원 / 하한 370,000원 → 최소 16,650원 |
| 건강보험 | 과세급여 × 3.545% | 소수점 이하 10원 단위 절사 |
| 장기요양 | 건강보험료 × 12.95% | 소수점 이하 원 단위 절사 |
| 고용보험 | 과세급여 × 0.9% | 소수점 이하 원 단위 절사 |

> 연봉 입력 시 과세급여 = 연봉 ÷ 12로 월 단위 환산 후 계산

### 계산 — 소득세

- 국세청 **근로소득 간이세액표** 기준 (월 급여 + 부양가족 수 기반)
- 간이세액표 데이터를 앱 내 상수로 내장 (네트워크 불필요)
- 지방소득세 = 소득세 × 10%

### 계산 — 실수령액

```
실수령액(월) = 과세급여 − 국민연금 − 건강보험 − 장기요양 − 고용보험 − 소득세 − 지방소득세
```

- 연봉 모드: 실수령액(월) 메인 표시 + 연 환산 = 실수령액(월) × 12
- 월급 모드: 실수령액(월) 메인 표시 + 연 환산 = 실수령액(월) × 12

### 표시

- 모든 금액: 소수점 없이 원 단위 정수, 3자리 콤마 구분
- 공제 내역 6개 항목 항상 표시 (접기/펼치기 없음)
- 입력값 0: 결과 및 공제 항목 모두 `—` 표시

## 데이터 관리 전략

### 전체 구조

세율 데이터는 **모두 Firestore에 저장**하고 앱이 실행 시 읽어온다.
세율이 변경되어도 앱 재배포 없이 Firestore 업데이트만으로 즉시 반영된다.

```
Firebase Functions (Scheduled)         수동 확인 후 Firebase 콘솔
        ↓                                        ↓
간이세액표 자동 갱신                    4대보험 요율 6개 수동 업데이트
        ↓                                        ↓
        └──────────── Firestore ─────────────────┘
                           ↓
                     앱 실행 시 읽기
                   (앱 재배포 불필요)
```

### Firestore 컬렉션 구조

```
tax_rates/
  └── latest/
        ├── nationalPensionRate      // 국민연금 요율
        ├── nationalPensionMin       // 국민연금 하한
        ├── nationalPensionMax       // 국민연금 상한
        ├── healthInsuranceRate      // 건강보험 요율
        ├── longTermCareRate         // 장기요양 요율
        ├── employmentInsuranceRate  // 고용보험 요율
        ├── incomeTaxTable           // 소득세 간이세액표 (2D 배열)
        ├── basedYear                // 기준 연도 (화면 표시용)
        └── updatedAt                // 마지막 업데이트 시각
```

## 데이터 갱신 방법

### 자동 갱신 — 소득세 간이세액표

Firebase Scheduled Function이 주기적으로 국세청 Excel을 내려받아 파싱 후 Firestore에 저장한다.
환율 계산기의 `scheduledExchangeRateRefresh` 패턴과 동일하게 구현한다.

```
scheduledTaxRateRefresh (연 1회, 매년 1월)
  ↓
국세청 간이세액표 Excel 다운로드
  ↓
파싱 → 2D 배열 변환
  ↓
Firestore tax_rates/latest 업데이트
```

### 수동 갱신 — 4대보험 요율 6개

자동화가 불가능한 데이터로, 변경 시 **Firebase 콘솔에서 직접 수정**한다.

| 항목 | 변경 주기 | 확인 방법 |
|------|-----------|-----------|
| 국민연금 요율 | 거의 변경 없음 | 국민연금공단 공지사항 확인 |
| 국민연금 기준소득월액 상한 | **매년 7월** 변경 | 국민연금공단 공지사항 확인 |
| 국민연금 기준소득월액 하한 | **매년 7월** 변경 | 국민연금공단 공지사항 확인 |
| 건강보험 요율 | **매년 1월** 변경 | 국민건강보험공단 보도자료 확인 |
| 장기요양 요율 | **매년 1월** 변경 | 국민건강보험공단 보도자료 확인 |
| 고용보험 요율 | 변경 시 공고 | 고용노동부 고시 확인 |

#### 확인 방법 상세

**국민연금 (매년 7월 적용)**
```
국민연금공단 홈페이지 → 알림소식 → 공지사항
검색어: "기준소득월액 상한액 하한액"
```

**건강보험 · 장기요양 (매년 1월 적용)**
```
국민건강보험공단 홈페이지 → 알림·뉴스 → 보도자료
검색어: "건강보험료율" 또는 "장기요양보험료율"
```

**고용보험 (변경 시)**
```
고용노동부 홈페이지 → 뉴스·소식 → 보도자료
검색어: "고용보험료율"
```

> 요율 변경은 매년 연말~연초에 뉴스로 크게 보도되므로 놓치기 어렵다.
> 확인 후 Firebase 콘솔 → Firestore → `tax_rates/latest` 문서에서 해당 필드값 수정.

## 추후 고려 사항 (v2)

- **비과세 항목 설정**: 식대(월 20만원), 차량유지비 등 입력 → 과세 기준 급여 조정

- **요율 연도 선택**: 해마다 달라지는 요율을 연도별로 선택
