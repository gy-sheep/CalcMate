# 더치페이 계산기 기획서

> **Phase**: `Phase 7`
> **한 줄 설명**: 여럿이 함께 낸 금액을 빠르고 공정하게 나눠주는 계산기

---

## 개요

식사나 모임 자리에서 계산할 때, 금액을 나누는 상황은 크게 두 가지다.
하나는 총액을 인원수로 똑같이 나누는 경우이고, 다른 하나는 각자 주문한 게 달라서 개별로 계산해야 하는 경우다.

이 계산기는 두 상황 모두 부담 없이 처리할 수 있도록 **균등 분배**와 **개별 계산**, 두 가지 모드를 제공한다.
계산대 앞에서 쓰는 도구인 만큼, **빠른 입력 → 즉각 결과 → 쉬운 공유** 흐름을 최우선으로 설계한다.

---

## 화면 구성

### 전체 구조

날짜 계산기와 동일하게 **커스텀 탭바 + PageView** 구조로 구성한다.
AppBar 아래 탭바가 고정되고, 탭 선택 또는 스와이프로 두 모드를 전환한다.
탭 전환 시 각 모드의 입력 데이터는 초기화된다.

```
┌──────────────────────────────────┐
│  ←  더치페이                    │  ← AppBar (공유 버튼 없음 — 각 모드 내 배치)
├──────────────────────────────────┤
│  [ 균등 분배  |  개별 계산  ]   │  ← 커스텀 탭바 (스트레치 인디케이터)
├──────────────────────────────────┤
│                                  │
│         (모드별 콘텐츠)          │  ← PageView
│                                  │
└──────────────────────────────────┘
```

## 기능 요구사항

### 공통

- 화면 진입 시 기본 탭은 **균등 분배**로 시작한다
- 탭 전환 및 스와이프 전환 모두 지원한다
- 탭 전환 시 각 모드의 입력 데이터는 초기화된다
- 계산 결과는 **버튼 없이 실시간**으로 업데이트된다
- 화면 진입 시 `PlatformDispatcher.instance.locale.countryCode`로 국가를 감지한다
  - **KR**: 나머지 처리 표시 / 팁 추가 미표시
  - **그 외**: 팁 추가 표시 / 나머지 처리 미표시

---

### 균등 분배 모드

#### 입력

- 총 금액은 화면 하단 커스텀 숫자 키패드로 입력한다 (시스템 키보드 사용 안 함)
- 금액 필드 탭 시 키패드가 올라오고, 다른 영역 탭 시 내려간다
- 인원 수는 `[−]` / `[+]` 버튼으로만 조절한다 — 키보드 입력 없음
  - 최솟값: 2명 / 최댓값: 30명
  - 앱 진입 시 기본값: 2명
- `[00]` 버튼: 숫자 두 자리 추가 (10,000 입력 시 유용)
- `[X]` 버튼: 마지막 자리 삭제 (백스페이스)
- `[C]` 버튼: 전체 금액 초기화

#### 나머지 처리 (KR 전용)

- **국가가 KR일 때만** 표시한다
- 항상 드롭다운으로 단위를 선택할 수 있으며, 기본값은 **100원 단위**다
- 나머지 처리 단위 옵션:

  | 국가 | 옵션 |
  |------|------|
  | KR | 100원 단위 / 1,000원 단위 |
  | JP | 10엔 단위 / 100엔 단위 |
  | 그 외 (USD, EUR 등) | 해당 없음 (팁 방식 사용) |

- **계산 방식**: 나머지 인원(계산한 사람 제외)은 선택 단위로 **내림한 금액**을 내고, 계산한 사람은 총액에서 나머지 인원 걷은 금액을 뺀 나머지를 부담한다
  - 예: 38,300원 ÷ 5명, 100원 단위 선택
    - 나머지 4명 → 7,600원 (100원 단위 내림)
    - 계산한 사람 → 38,300 - (7,600 × 4) = **8,900원**

- **결과 표시**:
  - 나눠떨어지는 경우: 단일 금액만 표시
    ```
    1인당  7,660원
    ```
  - 나머지가 있는 경우: 두 줄로 구분 표시
    ```
    나머지 4명    7,600원
    계산한 사람   8,900원
    ```

#### 팁 추가 (해외 전용)

- **국가가 KR이 아닐 때만** 표시한다
- 팁 옵션 칩: `없음` / `10%` / `15%` / `20%` / `직접입력`
- 팁은 총 금액에 먼저 더한 후 인원수로 균등 분배한다
- `직접입력` 선택 시 팁 비율 입력 필드 활성화 (키패드로 입력)
- 해외의 경우 소수점 입력이 필요하므로 키패드에 `[00]` 대신 `[.]` 버튼을 표시한다

#### 결과 표시

- 총 금액과 인원이 모두 입력되어야 결과 표시 (하나라도 0이면 `—` 표시)
- 결과 금액은 크게 강조 표시

### 개별 계산 모드

#### 참여자 관리

- 기본으로 2명 (A, B)이 세팅된 상태로 시작
- `[+ 추가]` 버튼으로 참여자 추가 (최대 10명)
- 참여자 이름은 기본 A, B, C... 알파벳 순서로 자동 부여
- 참여자 영역 우측에 `[편집]` 버튼을 두고, 탭하면 각 탭에 `[x]` 버튼 노출
  ```
  [A][x] [B][x] [C][x] [+ 추가]  ← 편집 모드
  ```
- `[x]` 탭 → 해당 참여자 삭제
  - 담당 항목이 있는 경우 확인 다이얼로그 표시: "이 참여자의 항목 N개가 함께 삭제됩니다"
- 참여자 이름 수정: 편집 모드에서 탭 → 이름 입력 필드 활성화 (시스템 키보드)

#### 항목 입력

항목 목록 하단에 입력 행이 항상 고정 노출된다 (인라인 입력 방식).

```
────────────────────────────────
삼겹살    36,000   A · B
냉면      10,000   C
────────────────────────────────
[항목명..]  [  금액  ]  [+]   ← 고정 입력 행
[A] [B] [C]                   ← 담당자 선택 칩
```

- **항목명**: 탭 시 시스템 키보드 활성화. 비워두면 "항목 N"으로 자동 처리
- **금액**: 탭 시 커스텀 숫자 키패드 활성화
- **담당자**: 참여자 칩 다중 선택 — 2명 이상 선택 시 해당 항목 금액을 균등하게 나눠 각 인원에게 귀속
- **`[+]`** 버튼 탭 시 항목 목록에 추가 후 입력 행 초기화
  - 금액 미입력 또는 담당자 미선택 시 `[+]` 비활성화

#### 항목 목록

- 추가된 항목을 스크롤 리스트로 표시
- 각 항목에 담당 인원 표시 (예: `A · B · C`)
- 항목 탭 → 해당 항목 내용이 입력 행에 채워지고, `[+]` 자리에 `[삭제]` / `[수정]` 버튼 표시
  ```
  [항목명..]  [ 금액 ]  [삭제][수정]
  [A] [B] [C]
  ```
- `[수정]` 탭 → 항목 목록에 반영 후 입력 행 초기화
- `[삭제]` 탭 → 해당 항목 목록에서 제거 후 입력 행 초기화

#### 결과 영역

- 항목 목록 하단에 고정 표시 (스크롤되지 않음)
- 총 금액, 인원별 금액 표시
- 공유 항목이 있을 경우: 소수점 나머지는 마지막 인원에게 1원씩 귀속

### 공유 기능

#### 공유 흐름

```
[결과 공유] 버튼 탭
      ↓
바텀 시트 — 영수증 이미지 미리보기
(영수증 위젯만 RepaintBoundary로 감싸 캡처 대상으로 지정)
      ↓
[공유하기] 버튼 탭
      ↓
시스템 공유 시트 (카카오톡, 메시지, 클립보드 등)
```

#### 공유 버튼 위치

- 균등 분배 / 개별 계산 모두 결과 영역 하단에 `[결과 공유]` 버튼 배치

## 예외 / 엣지 케이스

| 상황 | 기대 동작 |
|------|-----------|
| 총 금액이 0인 상태에서 인원 조절 | 결과 영역에 `—` 표시, 계산 안 함 |
| 1인당 금액이 0원 (총액 0) | 결과 `0원` 표시 |
| 인원 수보다 나머지가 작을 때 | 나머지 해당 인원만 1원 추가 표시 |
| 인원 `[−]` 버튼이 2명에서 클릭 | 무반응 (버튼 비활성화 상태) |
| 인원 `[+]` 버튼이 30명에서 클릭 | 무반응 (버튼 비활성화 상태) |
| 개별 계산 — 항목의 담당자 미선택 | `[+ 항목 추가]` 비활성화, 안내 텍스트 표시 |
| 개별 계산 — 담당자 삭제 시 해당 담당 항목 있음 | "이 참여자의 항목 N개가 함께 삭제됩니다" 확인 다이얼로그 |
| 개별 계산 — 항목 없이 결과 조회 | 결과 영역에 `항목을 추가해 주세요` 안내 |
| 금액 최대치 초과 (999,999,999원 초과) | 더 이상 입력되지 않음 (진동 피드백) |
| 팁 직접입력 — 0% 또는 빈 값 | `없음`과 동일하게 처리 |

## 테마 / 디자인 방향

- **컨셉**: 아기자기하고 반짝이는 느낌 — 귀엽고 화사한 분위기
- **주요 색상**:
  - 배경: 부드러운 핑크~라벤더 그라디언트
  - 강조색 (버튼, 탭 인디케이터): 로즈골드 또는 코랄 핑크
  - 결과 금액 텍스트: 딥 로즈 또는 버건디
  - 영수증 카드: 아이보리 또는 크림 화이트
- **디자인 요소**:
  - 버튼 모서리는 충분히 둥글게 (pill 형태)
  - 탭바 인디케이터에 반짝임(shimmer) 또는 글로우 효과
  - 영수증 공유 이미지에 작은 스파클(✦) 장식 포인트
  - 참여자 칩은 파스텔 컬러로 각각 구분 (A=핑크, B=라벤더, C=민트 등)
- **전체 톤**: 밝고 가볍게 — 어둡거나 무거운 느낌 없이

- **나머지 처리 (KR)**: 계산한 사람이 나머지를 부담하는 구조. 나머지 인원은 내림한 금액을 냄
- **팁 추가 (해외)**: 나머지 처리 대신 팁 비율 옵션 제공. 총액에 팁을 더한 후 균등 분배
- **개별 계산의 공유 항목**: 여럿이 함께 시킨 메뉴(안주, 공유 요리)를 자연스럽게 처리하는 핵심 기능
