# 설정 화면 기획서

> **Phase**: `Phase 14`
> **한 줄 설명**: 앱 전반의 표시·동작 방식을 사용자가 직접 조정하는 설정 화면

---

## 개요

사용자가 CalcMate의 테마, 계산기 기본값, 피드백 등 앱 전반의 동작을 개인화할 수 있는 화면.
현재 하드코딩된 `ThemeMode.system`, BMI 단위계, 환율 기본 통화 등을 사용자가 직접 변경하고
SharedPreferences에 저장한다. 메인 화면 우상단 설정 드롭다운 → "설정" 항목을 통해 진입한다.

---

## 진입 경로

- 메인 화면 앱바 우상단 **⋮ 메뉴** → **설정**
- (향후 확장) 딥링크 / 온보딩 화면에서 직접 진입

---

### 계산기 관리 서브 화면

```
┌───────────────────────────────────┐
│  ← 계산기 관리                  │
├───────────────────────────────────┤
│  기본 계산기                  ●   │  Switch (ON)
│  ─────────────────────────────── │
│  환율 계산기                  ●   │
│  ─────────────────────────────── │
│  단위 변환기                  ○   │  Switch (OFF → 메인 미표시)
│  ─────────────────────────────── │
│  ...                              │
└───────────────────────────────────┘
```

> **UI 스타일**: 섹션 헤더는 보조 텍스트(회색), 행 높이 56dp, 구분선은 Divider(indent: 16).
> 전체 배경은 `ColorScheme.surface`, 행 배경은 `ColorScheme.surfaceContainerLowest`.

## 섹션별 기능 요구사항

### 1. 화면

#### 1-1. 다크 모드
- 선택지: **시스템 기준** (기본) / **라이트** / **다크**
- 탭 시 바텀시트(라디오 리스트)로 선택
- 선택 즉시 `MaterialApp.themeMode` 반영 (앱 재시작 불필요)
- `SharedPreferences` 키: `'theme_mode'` — 값: `'system'` | `'light'` | `'dark'`
- 현재 `main.dart`의 `ThemeMode.system` 하드코딩을 Provider로 교체

---

### 2. 메인 화면

#### 2-1. 계산기 관리
- 설정 화면에서 타일 탭 → **서브 화면**(`계산기 관리`)으로 푸시
- 서브 화면: 전체 계산기 목록(13개)을 Switch 토글로 표시
- 타일 우측 값: 현재 표시 중인 계산기 수 (예: `9개`)
- ON → 메인 화면 목록에 표시 / OFF → 메인 화면 목록에서 제거
- `SharedPreferences` 키: `'calc_entry_hidden'` — 숨긴 항목의 id 목록 (StringList)
  - 기본값: 빈 리스트 (전체 표시)
  - ON/OFF 변경 즉시 메인 화면에 반영
- **최소 1개 이상** 표시 상태 유지 (마지막 1개는 OFF 불가)
- `CalcModeEntry.isVisible` 필드 활용 (이미 모델에 존재)
- `MainScreenViewModel`에 `toggleVisibility(String id)` intent 추가
- 메인 화면 리스트는 `isVisible == true` 항목만 렌더링
- **메인 화면 카드 스와이프 숨기기와 동일 로직 공유** — 어느 쪽에서 변경해도 즉시 동기화
  - 메인 화면: 카드 좌측 스와이프 → "숨기기" 버튼 → `toggleVisibility` 호출
  - 설정: Switch OFF → `toggleVisibility` 호출
  - 숨긴 계산기를 다시 표시하려면 설정 > 계산기 관리에서만 가능 (메인 화면에는 복원 UI 없음)

---

### 3. 계산기 기본값

> **로케일 자동 감지 + 사용자 수동 재정의** 방식.
> 앱 최초 실행 시 기기 로케일을 기반으로 자동 설정되며, 사용자가 설정 화면에서 수동 변경하면
> 그 값이 SharedPreferences에 저장되어 이후 로케일과 무관하게 저장된 값을 우선 사용한다.

#### 3-1. 환율 기준 통화
- 환율 계산기의 **fromCode** 초기값을 결정
- 선택지: KRW, USD, EUR, JPY, CNY, GBP, AUD, CAD, CHF, HKD (10개 주요 통화)
- 탭 시 바텀시트(라디오 리스트 + 통화 코드/이름 표시)로 선택
- `SharedPreferences` 키: `'default_currency'`
  - 저장값 없으면 기기 로케일의 국가 코드로 통화 추론 (KR → KRW, US → USD, JP → JPY 등)
  - 지원하지 않는 국가 코드인 경우 KRW 사용
  - 사용자가 수동 변경하면 해당 값을 저장 후 이후 로케일 무시
- 환율 ViewModel 초기화 시 이 값을 읽어 `fromCode` 설정

#### 3-2. BMI 단위
- BMI 계산기의 단위계 초기값을 결정
- 선택지: **kg / cm**, **lb / in**
- 탭 시 바텀시트(라디오 리스트)로 선택
- `SharedPreferences` 키: `'bmi_unit_system'` — 값: `'metric'` | `'imperial'`
  - 저장값 없으면 기기 로케일 기반 자동 설정
    - 야드파운드 사용 국가 (US, LR, MM): `imperial`
    - 그 외 모든 국가: `metric`
  - 사용자가 수동 변경하면 해당 값을 저장 후 이후 로케일 무시
- 현재 BMI ViewModel 내부에서 직접 읽는 동일 키 재사용 (하위 호환 유지)

---

### 4. 조작

#### 4-1. 버튼 진동 피드백
- 앱 전역 햅틱 피드백 ON/OFF
- Toggle 위젯 (기본값: ON)
- `SharedPreferences` 키: `'haptic_feedback'` — 기본값: `true`
- 드래그 정렬 햅틱(`HapticFeedback.lightImpact`) 및 향후 버튼 클릭 피드백에 적용

---

### 5. 앱 정보

#### 5-1. 버전 정보
- `package_info_plus` 패키지로 앱 버전 표시 (예: `1.0.0 (1)`)
- 탭 불가 (표시 전용 행)

#### 5-2. 오픈소스 라이선스
- Flutter 기본 제공 `LicensePage` 화면으로 푸시
- `showLicensePage(context: context, applicationName: 'CalcMate')`

#### 5-3. 개인정보처리방침
- 탭 시 외부 브라우저(`url_launcher`)로 개인정보처리방침 URL 열기

#### 5-4. 문의하기
- 탭 시 기본 이메일 앱으로 문의 메일 초안 열기 (`mailto:` 스킴)
- 수신자 이메일 주소는 `AppConfig` 상수로 관리

## SharedPreferences 키 정리

| 키 | 타입 | 기본값 | 기존 존재 여부 |
|----|------|--------|--------------|
| `theme_mode` | String | `'system'` | 신규 |
| `calc_entry_hidden` | StringList | `[]` | 신규 |
| `default_currency` | String | `'KRW'` | 신규 |
| `bmi_unit_system` | String | `'metric'` | 기존 (BMI ViewModel에서 사용) |
| `haptic_feedback` | bool | `true` | 신규 |

## 화면 전환

| 트리거 | 동작 |
|--------|------|
| 뒤로가기 / ← 버튼 | 메인 화면으로 Pop |
| 오픈소스 라이선스 탭 | `LicensePage` Push |
| 개인정보처리방침 탭 | 외부 브라우저 열기 |
| 문의하기 탭 | 이메일 앱 열기 |
| 계산기 관리 탭 | 서브 화면 Push |
| 다크 모드 / 환율 기준 통화 / BMI 단위 탭 | 바텀시트 열기 |

## 예외 / 엣지 케이스

| 상황 | 기대 동작 |
|------|-----------|
| SharedPreferences 읽기 실패 | 각 항목의 기본값 사용 |
| 계산기 관리 1개만 남은 상태에서 OFF 시도 | Switch 비활성화 또는 스낵바 안내 ("최소 1개 이상 표시해야 합니다") |
| 시스템 테마 변경 후 `시스템 기준` 선택 중 | 즉시 시스템 테마로 반영 |
| `bmi_unit_system` 키가 이미 BMI ViewModel에 저장된 값 | 설정 화면에서 해당 값 그대로 읽어 표시 |
| `url_launcher` 미설치 / 이메일 앱 없음 | 예외 무시 또는 스낵바 안내 ("앱을 열 수 없습니다") |
| 버전 정보 로드 실패 | `'-'` 표시 |

## 미포함 항목 (추후 고려)

- **계산 히스토리 설정**: Phase 14 히스토리 기능 개발 시 추가 (`history_enabled`, `history_limit`)
- **알림 설정**: 현재 알림 기능 없음

- **언어 설정**: 현재 기기 로케일 자동 적용 중, 다국어 지원 시 추가
- **메인 화면 순서 편집**: 이미 메인 화면 ⋮ 메뉴에 독립 기능으로 존재
