# 할인 계산기 기획서

> **Phase**: `Phase 8`
> **한 줄 설명**: 원가와 할인율을 입력하면 최종가와 절약액을 즉각 보여주는 계산기

---

## 개요

쇼핑 중 "이거 30% 할인이면 얼마야?"를 빠르게 계산하는 게 핵심 목적이다.
사용자는 대부분 매장이나 쇼핑 앱을 보는 중에 이 계산기를 열기 때문에, 입력이 최소화되고 결과가 즉각적으로 보여야 한다.
한국 쇼핑 환경에서는 "20% 세일 + 추가 10% 쿠폰"처럼 할인이 중첩되는 경우가 흔하므로 중첩 할인도 지원한다.

## 기능 요구사항

### 입력

- 숫자 키패드로만 입력 (시스템 키보드 사용 안 함)
- 원가는 천 단위 쉼표 자동 포맷 (50000 → 50,000)
- 활성화된 입력 필드는 테두리 강조로 구분
- 필드 탭 시 해당 필드로 포커스 이동

### 빠른 할인율 선택 칩

- 제공 값: 5%, 10%, 20%, 30%, 50%
- 칩 선택 시 할인율 입력 필드에 즉시 반영
- 직접 입력 시 칩 선택 해제
- 입력값과 칩 값이 일치하면 해당 칩 활성 표시

### 중첩 할인

- "추가 할인 쌓기" 버튼으로 두 번째 할인율 입력란 추가
- 최대 3단계까지 중첩 가능
- 계산 방식: 순차 적용 (복리 방식) — 50,000 × 0.7 × 0.9 = 31,500
- 결과 영역에 실질 할인율 표시 (예: "37% (30% + 10%)")
- 추가 할인 개별 제거 가능

### 결과

- 입력값 변경 즉시 계산 (별도 계산 버튼 없음)
- 원가·할인율 중 하나라도 미입력이면 결과 미표시
- 최종가를 가장 크고 강조된 텍스트로 표시
- 절약액은 강조색으로 표시

### 통화 기호 자동 표시

- 기기 로케일(`PlatformDispatcher.instance.locale.countryCode`)을 읽어 통화 기호를 결정한다
- 기호가 표시되는 위치 (총 5곳):
  1. 원가 입력 필드 앞
  2. 결과 카드 — 원가 취소선 텍스트 (`$50,000 →`)
  3. 결과 카드 — 최종가 행 텍스트 (`→ $31,500`)
  4. 결과 카드 — 절약액 수치 (`-$18,500`)
  5. 결과 카드 — 최종가 대형 수치 (`$31,500`)
- 매핑 테이블 (부가세 계산기의 `kCountryTaxRates`와 동일한 방식):

| 국가 코드 | 통화 기호 |
|-----------|----------|
| KR | ₩ |
| US, CA | $ |
| JP, CN | ¥ |
| GB | £ |
| DE, FR, IT, ES, NL 등 유로존 | € |
| IN | ₹ |
| AU | A$ |
| 그 외 | (기호 없음) |

- 매핑에 없는 국가는 기호를 표시하지 않는다 (숫자만 표시)
- 구현 시 `_getCurrencySymbol()` 형태의 private 헬퍼 함수로 분리한다 (파라미터 없이 `PlatformDispatcher.instance.locale.countryCode`를 직접 읽음)

### 키패드

- 숫자 0~9, 00, 쉼표(,), 소수점(.), 삭제(⌫), 초기화
- 원가 필드 활성화 시 소수점(.) 비활성화
- 할인율 필드 활성화 시 소수점(.) 활성화 (예: 12.5%)
- ⌫: 마지막 자리 한 글자 삭제
- 초기화: 현재 활성 필드 값 전체 삭제

## 예외 / 엣지 케이스

| 상황 | 기대 동작 |
|------|-----------|
| 할인율 100% 이상 입력 시도 | 99.9%로 제한 |
| 원가 0 입력 | 결과 미표시 |
| 중첩 할인 합산으로 최종가 0 이하 | 마지막 추가 할인율을 제한하여 최종가 0 미만 방지 |
| 소수점 포함 할인율 (예: 12.5%) | 정상 계산 |
| 매우 큰 금액 (10억 이상) | 정상 계산, 쉼표 포맷 그대로 표시 |

## 참고

- 중첩 할인 계산: 순차 적용 (복리 방식), 단순 합산 아님
  - 30% + 10% = 실질 37% (단순 합산 40%와 다름)
  - 계산식: 원가 × (1 - 0.3) × (1 - 0.1)
