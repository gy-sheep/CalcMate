# Q0023: 광고 삽입을 위한 레이아웃 예약 시점

## 결론

**지금 미리 플레이스홀더로 레이아웃 공간을 예약하는 것이 낫다.**

---

## 왜 지금 해야 하는가

Flutter에서 광고(보통 `google_mobile_ads` 패키지)는 **고정 높이의 배너 위젯**을 SafeArea 하단에 삽입하는 구조다.

나중에 추가하면 아래 문제가 발생한다:

- 계산기 화면마다 `Column` / `Stack` 구조를 일일이 수정해야 함
- `Scaffold` body 레이아웃이 광고 높이(50dp)만큼 밀리며 버튼 배치가 흐트러짐
- 기본 계산기처럼 화면 비율 기반으로 버튼 크기를 계산하는 화면은 특히 민감

---

## 권장 방법: 플레이스홀더 위젯 삽입

지금 당장 `google_mobile_ads` 패키지를 붙이지 않아도 된다.
**레이아웃 공간만 예약**해 두면 나중에 실제 광고 위젯으로 교체만 하면 된다.

### 1. 플레이스홀더 위젯 생성

```dart
// lib/core/widgets/ad_banner_placeholder.dart
class AdBannerPlaceholder extends StatelessWidget {
  const AdBannerPlaceholder({super.key});

  static const double bannerHeight = 50.0; // AdSize.banner 기준

  @override
  Widget build(BuildContext context) {
    // 출시 전 실제 BannerAd 위젯으로 교체
    return const SizedBox(height: bannerHeight);
  }
}
```

### 2. 각 계산기 Scaffold에 삽입

```dart
Scaffold(
  appBar: ...,
  body: Column(
    children: [
      Expanded(child: /* 계산기 본문 */),
      const AdBannerPlaceholder(), // 하단 고정
    ],
  ),
)
```

### 3. 실제 광고 연동 시 교체 (출시 직전)

```dart
// AdBannerPlaceholder.build()를 아래로 교체
return SizedBox(
  height: bannerHeight,
  child: AdWidget(ad: _bannerAd),
);
```

---

## 작업 범위

현재 구현된 계산기 6개 + 홈 화면에 동일한 패턴으로 일괄 삽입.
변경 패턴이 동일하므로 한 번에 처리 가능하다.

---

## 무료/유료 버전 연계

- **무료**: `AdBannerPlaceholder`가 실제 광고 위젯으로 동작
- **유료**: `AdBannerPlaceholder`를 `SizedBox.shrink()`(높이 0)로 조건 분기

```dart
class AdBannerPlaceholder extends StatelessWidget {
  const AdBannerPlaceholder({super.key});

  @override
  Widget build(BuildContext context) {
    final isPremium = ref.watch(premiumProvider); // 추후 연결
    if (isPremium) return const SizedBox.shrink();
    return const SizedBox(height: 50.0); // 또는 실제 BannerAd
  }
}
```

이 구조로 만들어두면 무료/유료 분기 작업(Q0020)과도 자연스럽게 연계된다.
