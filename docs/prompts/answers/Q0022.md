# Q0022: 디자인 통일성 점검

> **[2026-03-10 업데이트]** 이 문서의 해결 방향은 `AppTokens` 단일 클래스 기반이었으나, 이후 `Cm*` 컴포넌트 토큰 체계로 전면 교체되었습니다. 현행 토큰 구조는 `docs/conventions/UI_TOKEN_CONVENTION.md` 및 `lib/core/theme/app_design_tokens.dart` 참조.

**날짜**: 2026-03-07
**원본**: [PROPMTS.md](../PROPMTS.md#q0022)

## 질문

1. 각 개별 화면마다 폰트, 폰트 사이즈 등의 항목이 일관성이 없음
2. 화면의 특성이 다르더라도, 타이틀, 힌트, 섹션 타이틀, 컨텐츠 등의 텍스트들은 앱 전체에서 고유한 통일성이 필요해 보임
3. 위젯의 경우도 유사하거나 동일한 기능을 하는 경우(ex. input text) 각 계산기마다 전혀 다른 스타일이 아닌 앱 전체에 일관된 경험을 제공해야함

## 답변

### 현황 분석

현재 `AppTokens`(`app_design_tokens.dart`)에 토큰이 잘 정의되어 있지만, **실제 활용률이 낮습니다.**

| 구분 | 토큰 사용 | 하드코딩 |
|------|----------|---------|
| AppBar 타이틀 | ✅ | - |
| AppSegmentControl / TabBar | ✅ | - |
| 버튼 텍스트 (키패드) | - | ✅ fontSize: 22/28 |
| 디스플레이 결과값 | - | ✅ fontSize: 40~80 |
| 카드 내 타이틀/값 | - | ✅ 각기 다름 |
| TextField | 일부 ✅ | 일부 하드코딩 |

---

### 핵심 문제

#### 1. 텍스트 역할(Role)에 따른 스타일 정의 없음

현재 토큰은 **숫자값** 만 있습니다 (`fontSizeBody = 14`, `fontSizeLabel = 12`). 하지만 실제 화면에서 필요한 것은 **의미(semantic) 기반 스타일**입니다.

```
// 현재: 값만 정의
fontSizeBody = 14

// 필요한 것: 역할별 TextStyle 정의
textStyleSectionTitle   → 타이틀 역할
textStyleHint           → 힌트/설명 역할
textStyleContentValue   → 결과 값 역할
textStyleCaption        → 부연 설명 역할
```

#### 2. Feature마다 같은 역할인데 다른 스타일

예: 날짜/나이/할인 계산기 모두 "메인 결과값"을 표시하지만

```dart
// 나이 계산기
TextStyle(fontSize: 56, fontWeight: FontWeight.w800)

// 날짜 계산기
TextStyle(fontSize: 22, fontWeight: FontWeight.w600)

// 부가세 계산기
TextStyle(fontSize: 40, fontWeight: FontWeight.w300)
```

#### 3. 입력 위젯(TextField) 스타일 불일치

더치페이는 `AppTokens.fontSizeBody`를 쓰고, 대출 계산기 프로토타입은 `fontSize: 28`을 직접 씁니다. `InputDecoration` 구성도 화면마다 다릅니다.

---

### 해결 방향

#### 1단계: AppTokens에 Semantic TextStyle 추가

`app_design_tokens.dart`에 TextStyle 수준의 토큰 추가:

```dart
class AppTokens {
  // 기존 숫자 토큰 유지...

  // --- Semantic TextStyle (새로 추가) ---

  /// 섹션 타이틀 (ex. "기준 금액", "계산 결과")
  static const textStyleSectionTitle = TextStyle(
    fontSize: 13,
    fontWeight: FontWeight.w600,
    letterSpacing: 0.3,
  );

  /// 입력 힌트 / 안내 문구
  static const textStyleHint = TextStyle(
    fontSize: 14,
    fontWeight: FontWeight.w400,
  );

  /// 본문 / 일반 콘텐츠
  static const textStyleBody = TextStyle(
    fontSize: 14,
    fontWeight: FontWeight.w400,
  );

  /// 부연 설명 / 캡션
  static const textStyleCaption = TextStyle(
    fontSize: 12,
    fontWeight: FontWeight.w400,
  );

  /// 결과값 — Large (기본 계산기, 나이 등 핵심 숫자)
  static const textStyleResultLarge = TextStyle(
    fontSize: 56,
    fontWeight: FontWeight.w300,
    height: 1.1,
  );

  /// 결과값 — Medium (부가세, 더치페이 금액 등)
  static const textStyleResultMedium = TextStyle(
    fontSize: 32,
    fontWeight: FontWeight.w300,
    letterSpacing: -0.5,
  );

  /// 결과값 — Small (보조 수치, 환율 등)
  static const textStyleResultSmall = TextStyle(
    fontSize: 20,
    fontWeight: FontWeight.w400,
  );
}
```

> **색상은 포함하지 않습니다.** 각 화면 고유의 테마 색상(kAgeAccent, kVatReceiptText 등)은 `.copyWith(color: ...)` 로 덮어씁니다.

```dart
// 사용 예시
Text('세', style: AppTokens.textStyleResultSmall.copyWith(color: kAgeAccent))
```

---

#### 2단계: 공통 입력 위젯 만들기

현재 각 화면의 `TextField`가 비슷하게 생겼지만 각기 다르게 정의됩니다. `AppTextField`를 공통 위젯으로 추출합니다.

```dart
// lib/presentation/widgets/app_text_field.dart
class AppTextField extends StatelessWidget {
  final TextEditingController controller;
  final String? hintText;
  final Color textColor;
  final Color hintColor;
  final TextInputType keyboardType;
  final FocusNode? focusNode;
  final VoidCallback? onTap;

  @override
  Widget build(BuildContext context) {
    return TextField(
      controller: controller,
      focusNode: focusNode,
      keyboardType: keyboardType,
      style: AppTokens.textStyleBody.copyWith(color: textColor),
      decoration: InputDecoration(
        hintText: hintText,
        hintStyle: AppTokens.textStyleHint.copyWith(color: hintColor),
        border: InputBorder.none,
        isDense: true,
        contentPadding: EdgeInsets.zero,
      ),
    );
  }
}
```

---

#### 3단계: 일관성 가이드라인 정의

텍스트 역할별 사용 규칙:

| 역할 | 토큰 | 예시 |
|------|------|------|
| 화면/섹션 타이틀 | `textStyleSectionTitle` | "기준 금액", "참여자 목록" |
| 입력창 힌트 | `textStyleHint` | "금액을 입력하세요" |
| 일반 내용 | `textStyleBody` | 목록 항목, 설명 |
| 보조 설명 | `textStyleCaption` | 단위, 날짜 포맷 |
| 핵심 결과값 | `textStyleResultLarge/Medium/Small` | 계산 결과 숫자 |

색상은 각 화면의 color 파일에서 `.copyWith(color: ...)` 로 주입. 폰트 크기/굵기/높이는 AppTokens 토큰만 사용.

---

### 작업 우선순위 제안

1. **AppTokens에 Semantic TextStyle 추가** — 파일 1개 수정, 기반 작업
2. **공통 AppTextField 위젯 생성** — 파일 1개 추가
3. **화면별 마이그레이션** — 각 화면 위젯에서 하드코딩 → 토큰으로 교체
4. **CalcModeCard 등 공통 위젯 내 하드코딩 정리**

마이그레이션은 새 기능 개발 때 해당 화면을 수정하면서 자연스럽게 진행하거나, 별도 리팩토링 태스크로 한 번에 진행할 수 있습니다.
