# Q0022: 디자인 통일성 점검

**날짜**: 2026-03-04
**원본**: [PROPMTS.md](../PROPMTS.md#q0022)

## 질문

1. 각 개별 화면마다 폰트, 폰트 사이즈 등의 항목이 일관성이 없음
2. 화면의 특성이 다르더라도, 타이틀, 힌트, 컨텐츠 등의 텍스트들은 앱 전체에서 고유한 통일성이 필요해 보임
3. 비슷한 컨테이너의 사이즈
4. 이외에 더 통일성의 유지가 필요한 항목들이 있는지 점검 필요함

## 답변

### 현재 상황 요약

각 화면이 `_k*` 패턴으로 색상/크기 상수를 **화면마다 독립적으로 선언**하고 있습니다. 화면 특성이 달라 배경 테마는 달라도 괜찮지만, 공통으로 통일돼야 할 항목들이 일치하지 않습니다.

---

### 1. 불일치 항목 분석

#### AppBar 타이틀 폰트 웨이트
| 화면 | fontSize | fontWeight |
|------|----------|-----------|
| 기본 계산기 | 18 | w600 |
| 환율 계산기 | 18 | w600 |
| 단위 변환기 | 18 | w600 |
| 부가세 계산기 | 18 | w600 |
| 나이 계산기 | 18 | **w700** ← 혼자 다름 |

#### 카드 borderRadius
| 화면 | 주요 컨테이너 borderRadius |
|------|--------------------------|
| 단위 변환기 | 12 |
| 부가세 계산기 | 20 |
| 나이 계산기 | 16 |
| 환율 계산기 | 20 |

#### 결과값 폰트 크기
| 화면 | 결과값 fontSize | fontWeight |
|------|--------------|-----------|
| 기본 계산기 | 56 | w300 |
| 환율 계산기 | 28 | w300 |
| 부가세 계산기 | 40 | w300 |
| 나이 계산기 | 56 | **w800** ← 웨이트 다름 |

#### 카드 padding
| 화면 | 내부 padding |
|------|-------------|
| 부가세 계산기 | 20~28 혼재 |
| 환율 계산기 | 16~24 혼재 |
| 나이 계산기 | 14~20 혼재 |

---

### 2. 이미 잘 통일된 항목

- **AppBar 구조**: 뒤로가기 아이콘(`Icons.arrow_back_ios`, size 20), 아이콘 박스(28×28, borderRadius 7), Hero 애니메이션 — 전 화면 동일
- **Divider**: `Color(0x55FFFFFF)`, thickness 0.5~1.5 — 거의 동일
- **토스트 알림**: borderRadius 20, 패딩 구조 — 동일
- **기능 버튼(AC 등)**: `Color(0xCCFFFFFF)` — 동일

---

### 3. 추가로 통일이 필요한 항목

| 항목 | 현황 | 권장 |
|------|------|------|
| 세그먼트 컨트롤 스타일 | 화면마다 재구현 | 공통 위젯으로 추출 |
| 소수점 표시 자릿수 정책 | 화면마다 다름 | 카테고리별 정책 통일 |
| 빈 상태(empty state) UI | 일부 없음 | 통일 필요 |
| 로딩 인디케이터 | 환율만 `CupertinoActivityIndicator` | 공통 정의 필요 |
| 에러 메시지 표시 방식 | 토스트/없음 혼재 | 통일 필요 |

### 4. 권장 해결 방법

`lib/core/theme/` 아래에 **디자인 토큰 파일**을 추가하는 것이 가장 효과적입니다.

```
lib/core/theme/
├── app_theme.dart         ← 기존 (Material 테마)
└── app_design_tokens.dart ← 신규 추가
```

```dart
// app_design_tokens.dart
abstract class AppTokens {
  // ── Typography ──────────────────────────
  static const double fontSizeAppBar     = 18;
  static const FontWeight weightAppBar   = FontWeight.w600;
  static const double fontSizeLabel      = 12;
  static const double fontSizeBody       = 14;
  static const double fontSizeValue      = 16;

  // ── Shape ───────────────────────────────
  static const double radiusCard         = 16;
  static const double radiusChip         = 20;
  static const double radiusTag          = 6;
  static const double radiusAppBarIcon   = 7;

  // ── Spacing ─────────────────────────────
  static const double paddingScreenH     = 16;
  static const double paddingCardInner   = 16;
  static const double paddingAppBarH     = 16;
  static const double paddingAppBarV     = 12;

  // ── Component ───────────────────────────
  static const double sizeAppBarIcon     = 28;
  static const double heightButton       = 64;
  static const double heightSegment      = 36;
}
```

### 5. 작업 우선순위 제안

전면 리팩토링보다 **신규 화면부터 토큰 적용 → 기존 화면 점진 교체** 순서가 현실적입니다.

| 우선순위 | 작업 | 이유 |
|---------|------|------|
| 1 | `app_design_tokens.dart` 파일 생성 | 기준 정의 |
| 2 | AppBar 타이틀 fontWeight 통일 (나이 계산기 w700 → w600) | 변경 범위 최소 |
| 3 | 카드 borderRadius → `AppTokens.radiusCard(16)` 통일 | 시각적 통일감 즉시 체감 |
| 4 | 세그먼트 컨트롤 공통 위젯 추출 | 4개 화면 재사용 |
| 5 | 기존 화면 점진적 토큰 교체 | 기능 영향 없이 진행 가능 |
