# Q0008: 코드내에서 사용중인 문자열이나 사이즈값 관리

**날짜**: 2026-02-27
**원본**: [PROPMTS.md](../PROPMTS.md#q0008)

## 질문

1. 안드로이드 개발 시에는 문자열이나, 색상값, 폰트 사이즈, 위젯 사이즈 등을 xml로 관리했었는데, 플러터에서는 어떻게 관리하지
2. 특히 문자열의 경우 다국어 지원을 어떻게 처리하지

## 답변

### 1. Flutter의 리소스 관리 방식

Android의 XML 리소스와 Flutter의 대응 관계:

| Android XML | Flutter |
|-------------|---------|
| `strings.xml` | ARB 파일 (`l10n/`) 또는 Dart 상수 클래스 |
| `colors.xml` | `ThemeData` + `ColorScheme` (권장) 또는 상수 클래스 |
| `dimens.xml` | Dart 상수 클래스 |
| `styles.xml` | `ThemeData`, `TextTheme` |

---

### 색상 — `ThemeData` 활용 (권장)

CalcMate는 이미 `app_theme.dart`에 `ThemeData`가 설정되어 있으므로, 색상은 `Theme.of(context)`로 참조하는 방식이 Flutter 표준이다.

```dart
// 사용 예
Container(color: Theme.of(context).colorScheme.primary)
```

커스텀 색상이 필요한 경우 상수 클래스로 분리한다.

```dart
// core/constants/app_colors.dart
class AppColors {
  static const cardOverlayStart = Color(0x99000000); // 60% black
  static const cardOverlayEnd   = Color(0x33000000); // 20% black
}
```

### 사이즈 / 여백 — 상수 클래스

```dart
// core/constants/app_sizes.dart
class AppSizes {
  static const cardBorderRadius = 16.0;
  static const cardPadding      = 20.0;
  static const cardSpacing      = 16.0;
  static const iconSize         = 20.0;

  static const fontTitle        = 18.0;
  static const fontDescription  = 13.0;
}
```

```dart
// 사용 예 (calc_mode_card.dart)
BorderRadius.circular(AppSizes.cardBorderRadius)
Padding(padding: EdgeInsets.all(AppSizes.cardPadding))
```

### 문자열 + 다국어 — flutter_localizations (ARB)

Flutter 공식 방식은 `ARB(Application Resource Bundle)` 파일이다.

**설정 순서:**

**① `pubspec.yaml`**
```yaml
dependencies:
  flutter_localizations:
    sdk: flutter
  intl: ^0.20.0

flutter:
  generate: true          # 자동 코드 생성 활성화
```

**② ARB 파일 생성**
```
lib/
└── l10n/
    ├── app_ko.arb   # 한국어
    └── app_en.arb   # 영어
```

```json
// lib/l10n/app_ko.arb
{
  "appName": "CalcMate",
  "basicCalculator": "기본 계산기",
  "basicCalculatorDesc": "사칙연산 및 공학 계산",
  "exchangeRate": "환율 계산기",
  "exchangeRateDesc": "실시간 전 세계 환율 변환"
}
```

```json
// lib/l10n/app_en.arb
{
  "appName": "CalcMate",
  "basicCalculator": "Basic Calculator",
  "basicCalculatorDesc": "Arithmetic and scientific calculations",
  "exchangeRate": "Exchange Rate",
  "exchangeRateDesc": "Real-time global currency conversion"
}
```

**③ `main.dart` 설정**
```dart
MaterialApp(
  localizationsDelegates: AppLocalizations.localizationsDelegates,
  supportedLocales: AppLocalizations.supportedLocales,
  ...
)
```

**④ 사용**
```dart
// flutter pub run intl:generate 후 자동 생성된 클래스 사용
Text(AppLocalizations.of(context)!.basicCalculator)
```

### CalcMate 프로젝트 적용 구조

```
lib/
├── core/
│   ├── constants/
│   │   ├── app_colors.dart   # 커스텀 색상
│   │   └── app_sizes.dart    # 여백/사이즈
│   └── theme/
│       └── app_theme.dart    # ThemeData (기존)
└── l10n/
    ├── app_ko.arb            # 한국어 (기본)
    └── app_en.arb            # 영어
```

> **우선순위**: CalcMate가 한국어 단일 출시를 목표로 한다면 ARB 설정은 Phase 14(마무리)에서 처리하고, 현재는 `app_sizes.dart`, `app_colors.dart` 상수 클래스 분리부터 진행하는 게 효율적이다.
