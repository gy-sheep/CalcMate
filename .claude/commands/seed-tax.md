Firestore 세율 데이터를 갱신한다. 아래 순서를 따른다.

## 1단계: 4대보험 요율 수집 요청

아래 표를 보여주고, 값은 비워둔 채로 사용자에게 입력을 요청한다.

```
| 항목                        | 값 |
|-----------------------------|-----|
| 국민연금 요율               |     |
| 국민연금 하한액             |     |
| 국민연금 상한액             |     |
| 건강보험 요율               |     |
| 장기요양 요율 (건보료 대비) |     |
| 고용보험 요율               |     |
```

표 아래에 다음 안내를 추가한다:

> Gemini 등을 통해 올해 최신 4대보험 요율을 확인한 뒤 위 표의 값을 채워주세요.
> 검색 키워드 예시: "2026년 4대보험 요율", "국민연금 기준소득월액 상하한"

**사용자가 값을 입력할 때까지 다음 단계로 진행하지 않는다.**

## 2단계: 변경 여부 확인

`functions/scripts/seed_tax_rates.ts`의 `INSURANCE_RATES` 상수를 읽어 사용자가 입력한 값과 비교한다.

- **변경된 항목이 있으면**: 변경 내용을 표로 보여주고(현재값 → 새값), 3단계로 진행한다.
- **모든 값이 동일하면**: "요율 변경 없음"을 알리고, 간이세액표 Excel도 갱신할지 사용자에게 확인한다.
  - 사용자가 "아니오"라고 하면 스크립트를 실행하지 않고 종료한다.
  - 사용자가 "예"라고 하면 3단계로 진행한다.

## 3단계: 간이세액표 Excel 요청

다음을 요청한다:

> 간이세액표 Excel 파일 경로를 알려주세요.
> 국세청 홈택스에서 "근로소득 간이세액표"로 검색하면 다운로드할 수 있습니다.

**사용자가 파일 경로를 제공할 때까지 다음 단계로 진행하지 않는다.**

## 4단계: 스크립트 업데이트 + 실행

1. 요율 값이 변경된 경우에만 `functions/scripts/seed_tax_rates.ts`의 `INSURANCE_RATES` 상수를 업데이트한다.
2. 시딩 스크립트를 실행한다:
   ```bash
   cd functions && npx ts-node scripts/seed_tax_rates.ts <사용자가 제공한 Excel 경로>
   ```
3. 실행 결과(행 수, 문서 크기, 요약)를 보여준다.
4. 실패 시 오류 원인을 분석하고 해결 방법을 안내한다.

## 5단계: 완료 확인

업로드 성공 시 아래 내용을 요약해서 보여준다:
- 기준 연도
- 4대보험 요율 6개
- 간이세액표 행 수
- 문서 크기
